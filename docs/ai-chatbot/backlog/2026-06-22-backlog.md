## 남은 작업

### 보안 (위생 / 잔여)
- **git 이력 재작성** — 과거 커밋의 `.env`/`target`/`node_modules`/`dist` 제거(`git filter-repo`). **파괴적·팀 조율 필요**(force-push·재클론). 단, 키 노출 위험 자체는 revoke로 **이미 해소**됨(위생/용량 목적). [이슈](../../_shared/issues/2026-06-21-git-history-secret-exposure.md).
- **AI 작업 브랜치 push/PR** — 보안 동결 해제 후 `feature/jeongheon-chatbot`(6커밋) push 및 PR.

### P1 — 안정성 / 품질
- **App.vue mojibake 복원** (`fix(front)`) — 깨진 한글 문자열 20개 복원 + UTF-8 재저장. [이슈 open](../../_shared/issues/2026-06-22-frontend-app-vue-mojibake-strings.md).
- **프론트 컴포넌트 테스트(Vitest)** — DOM/접근성/입력잠금. 팀 합의 후. [이슈](../../_shared/issues/2026-06-21-frontend-component-test-infra.md).
- **limiter 운영 확장** — Redis 분산 limiter, 사용자별 일일 한도, circuit breaker, 전체 요청 deadline, 429·지연·비용 지표/알림.
- **자동 임포트 견고화** — `PUBLIC_DATA_SERVICE_KEY` 미설정/만료 사전 감지, API 장애/지연 구분, 해당 월 데이터 없음 안내 세분화.
- **운영 견고화 — `SSAFY_GMS_API_KEY` 필수 (기동 차단 현상, 2026-06-22 확인)**
  - **현상**: 키 미설정 시 **백엔드 컨테이너 기동 실패**.
  - **원인**: `application.properties`의 `spring.ai.openai.api-key=${SSAFY_GMS_API_KEY:}`(빈 fallback)이지만 **Spring AI OpenAI 자동구성이 빈 키를 거부**해 컨텍스트 로드 실패. (증거: `HomeApplicationTest`가 `spring.ai.openai.api-key=context-test-key` 더미 키를 주입해야 부팅됨.)
  - **조치 옵션**: ① 필수 키로 명시 + 기동 전 친화적 검증/명확 메시지, ② AI 자동구성 조건부화(키 없으면 앱은 기동하고 `/api/ai/chat`만 친화 503), ③ `Backend/README`·`.env.example`에 "미설정 시 기동 실패" 명시. → 별도 이슈/troubleshooting 분리 검토.
