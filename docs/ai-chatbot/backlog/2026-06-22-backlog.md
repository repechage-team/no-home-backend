### P1 — 안정성 / 품질
- ✅ **App.vue mojibake 복원 (완료, frontend #2 머지)** — 깨진 한글 문자열 20개 복원 + UTF-8 재저장. [이슈](../../_shared/issues/2026-06-22-frontend-app-vue-mojibake-strings.md).
- ✅ **App.vue 삭제확인 불일치 (완료, frontend #3 머지 `3102786`)** — 템플릿 `placeholder` ↔ 비교값을 `'삭제'`로 통일. (현재 코드: `placeholder="삭제"` ↔ `deleteConfirm !== '삭제'` 일치 확인.)
- **프론트 컴포넌트 테스트(Vitest)** — DOM/접근성/입력잠금. 팀 합의 후. [이슈](../../_shared/issues/2026-06-21-frontend-component-test-infra.md).
- **limiter 운영 확장** — Redis 분산 limiter, 사용자별 일일 한도, circuit breaker, 전체 요청 deadline, 429·지연·비용 지표/알림.
- ✅ **자동 임포트 견고화 (완료, 팀원 cmsik1, backend #13 머지 `e683784`, 이슈 [#9](https://github.com/repechage-team/no-home-backend/issues/9) CLOSED)** — `resultCode` 파싱으로 무효/만료 키·오류를 분류(`PublicDataApiException`/`AutoImportException`), "데이터 없음" 위장 해소. 챗봇(`ai`) 소비자 측 문구 연동은 필요 시 후속. [이슈 문서](../../_shared/issues/2026-06-22-autoimport-error-categorization.md).
- ✅ **운영 견고화 — `SSAFY_GMS_API_KEY` graceful 처리 (완료, backend #7 머지)**
  - **해결(②)**: `AiKeyEnvironmentPostProcessor`가 키 미설정 시 OpenAI 모델 자동구성을 비활성화(`spring.ai.model.*=none`)해 **앱은 정상 기동**, `ChatClient` 빈 미생성 → `/api/ai/chat`은 친화 **503**(컨트롤러 `@Nullable ChatClient`). 기동 시 WARN 로그.
  - **무효/만료 키**: 런타임 401/403을 인증 실패로 분류해 친화 503 + WARN(원문·secret 미노출).
  - 문서: `Backend/README`·`.env.example`에 동작 명시. 테스트: 무키 컨텍스트 로드 + 컨트롤러 503/인증오류 단위테스트(전체 87건 그린). 브랜치 `feature/jeongheon-ai-key-resilience`.
