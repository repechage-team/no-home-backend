## 남은 작업

### 검색 패널 접힘(FE #14)의 AI 영향 검증 (✅ 완료)
팀원의 검색 패널 자동 접힘 도입 후 AI 에이전트 6개 동작을 검증 — 모두 Vue 데이터 기반이라 패널 접힘과 **로직 독립, 기능 문제 없음**. UX 보정 1건만 적용.
- ✅ **setFilters 패널 펼침 보정**(FE #15): `setFilters`는 `searchHouses` 미호출이라 패널 자동 접힘 로직을 안 타, 직전 검색으로 접혀 있으면 변경 필터가 가려졌다. `handleAgentCommand` setFilters 분기에 `searchPanelCollapsed=false` 추가. 브라우저 E2E(search→접힘 / setFilters→펼침 + 전세 탭 노출) 확인.

### P1 (신규 핵심) — 재설계 후속: 맥락 이해 ↔ 동작 괴리 해소
returnDirect 단일 액션 모델의 한계로, 단일 동작은 개선됐으나 결과 기반·순차 동작이 미해결(직접 사용 중 확인).
- ⬜ **② 결과 기반 선택** — "면적 가장 작은 매물 선택" 등은 현재 결과 목록(아파트명·면적·가격)이 모델에 전달되지 않아 환각(1번 선택). `AssistantRequest`에 현재 결과 요약(순번·아파트명·면적·가격)을 동봉하고 프롬프트에 노출 + **면적 정렬 옵션(sort)** 추가.
- ⬜ **③ 다중 동작** — "3페이지 이동 후 2번째 선택" 같은 순차 동작은 `@Tool(returnDirect=true)`가 1액션/호출이라 첫 동작만 실행. 해결: **(B) 수동 루프**(`internalToolExecutionEnabled(false)` + `ToolCallingManager`로 멀티 tool 수집·순차 적용) 또는 `AssistantResponse.commands[]` 배열 반환. 구조 변경 규모 큼 → 별도 PoC 권장.

### P1 — 안정성 / 품질
- **프론트 컴포넌트 테스트(Vitest)** — DOM/접근성/입력잠금. 팀 합의 후. [이슈](../../_shared/issues/2026-06-21-frontend-component-test-infra.md).
- **limiter 운영 확장** — Redis 분산 limiter, 사용자별 일일 한도, circuit breaker, 전체 요청 deadline, 429·지연·비용 지표/알림.

### P2 — 기능 확장 / 견고화
- **전월세 자동임포트 견고화** — 매매(`resultCode "000"`/UA 차단)와 동일 이슈가 전월세(`PublicDataAptRentImportService`)에도 있는지 점검·동일 처리. [트러블슈팅](../troubleshooting/2026-06-23-publicdata-resultcode-000-and-ua.md).
- **라이브 검색 영속 실패 관측** — `PublicDataBatchPersistService.persistAsync()`가 저장 실패를 `log.warn`만 하고 무시(silent) → 누락 시 동일 검색이 매번 라이브 API 재호출. 실패율/누락 관측 지표 추가.
- **질의 능력 확대** — 가격대/면적/연식, 기간·추세, 다중 구 비교 / 비서울 lawdCd 매핑 확장(현재 서울 25개 자치구만).
- **스트리밍 응답(선택)** — SSE/fetch 스트림. 첫 질의 지연 UX 보강.
