### 에이전트 Phase 2 (✅ 머지 완료)
[제안서](../proposals/2026-06-22-agent-mode.md) · [MVP 리포트](../reports/2026-06-22-agent-mode-mvp-implementation.md) · 구현 리포트(작성 예정 `../reports/2026-06-23-agent-mode-phase2-implementation.md`).
설계 원칙(MVP 계승): **capability = 단일 출처, 프론트가 최종 강제자.** 백엔드 프롬프트는 보조,
백엔드 가드는 레포 독립·안전 불변식(서울 지역·연월 범위·action allow-list)만 강제.

> Phase 2는 backend #18 / frontend #7로 **머지 완료**. 아래는 구현 요약(이력).

- ✅ **PR A — 필터** `sort · umdNm · minPrice · maxPrice` — **머지 완료**
  - 프론트 apply 경로 기존재(`filterSchema`·generic `applyAgentFilters`·`buildHouseSearchRequests`),
    백엔드 검색도 4필드 지원 → **백엔드 검색 코드 변경 0**. 백엔드는 `SYSTEM_PROMPT_TEMPLATE` 값 포맷 안내만.
  - 프론트 `buildAgentSummary`에 가격대·실효 정렬 표기 추가(UI 결합은 normalize 기존 처리로 정합).
  - **필터 값 가드 무추가**(프론트 최종 강제). 백엔드 `AgentCommandGuardsTest` passthrough 1건.
  - 검증: 백엔드 113 / 프론트 44 그린 + build. 상세 [Phase 2 리포트](../reports/2026-06-23-agent-mode-phase2-implementation.md).
- ✅ **PR B — 액션** `paginate · mapFocus · selectItem` — **머지 완료**
  - 예약 필드 `page · direction · itemIndex` 첫 사용. 백엔드 `ALLOWED_ACTIONS`+가드+프롬프트+`totalPages` 주입.
  - 프론트 신규 순수 모듈 `chat/agentActions.js`(`resolvePaginateTarget`·`resolveItemTarget`) + `handleAgentCommand` 3 case
    (기존 `searchHouses`/`selectItem`/`focusMapItem` 재사용).
  - **인덱싱**: 1-based → 프론트 0-based 변환. **상한**: 서버 하한만(page≥1·itemIndex≥1), 프론트 라이브 clamp.
  - 검증: 가드 7건·액션 13건 추가, 그린. mapFocus/selectItem 지도 표시는 브라우저 E2E 후속.
- ✅ **머지 완료** — backend [#18](https://github.com/repechage-team/no-home-backend/pull/18) / frontend [#7](https://github.com/repechage-team/no-home-frontend/pull/7).
- ⬜ **(흡수) `AiProviderErrors` 추출** — tool calling 재설계 Phase 1에서 공통화. `ai.agent.enabled` kill-switch는 `/assistant` 피처 플래그로 대체.
