6. ✅ **챗 에러 처리 표준화 (완료, 2026-06-21)** — 빈 입력은 HTTP 400, 중첩된 모델 타임아웃은 504, 그 외 모델·Tool 장애와 빈 모델 응답은 503 + `ApiResponse.fail`로 통일. `spring.ai.tools.throw-exception-on-error=true`로 Tool 예외를 Controller까지 전파하고 내부 예외 메시지는 응답에서 제거. 전체 테스트 총 60건 통과.
7. ✅ **입력·호출 제한 (완료, 2026-06-21)** — 메시지 최대 500자, 로그인 사용자별 1분당 10개 token을 연속 보충하는 token bucket 적용. 사용자당 동시 모델 요청은 1개로 제한해 중복 전송을 차단하고, token 부족은 HTTP 429 + 동적 `Retry-After`, 처리 중 중복 요청은 409, 인증 식별 누락은 401로 반환. 모델 HTTP 연결 3초·응답 25초, AI 재시도 최대 2회로 설정하고 모든 값은 환경변수로 재정의 가능. 프론트는 전송 완료→데이터 확인→지연 안내를 단계별로 표시하고 답변 중 입력창을 잠금. 전체 백엔드 테스트 68건, 프론트 테스트 3건·프로덕션 빌드 통과.
   - **사용자 시나리오:** [사용량 리미터 및 UX 사용자 시나리오](../reports/2026-06-21-usage-limiter-ux-scenarios.md)에 정상·중복·한도 소진·인증·장애·새로고침·접근성 흐름과 token 차감 기준을 정리.
   - **운영 확장 안전망(후속):** 다중 인스턴스용 Redis limiter, 사용자별 일일 한도, 공급자 장애 circuit breaker, 전체 요청 deadline, 429·타임아웃·지연·비용 지표 및 알림.
   - ✅ **내부 테스트용 리미터 토글 (완료, 2026-06-22)** — `AI_CHAT_RATE_LIMIT_ENABLED`(기본 `true`)가 `false`면 `AiChatRateLimiter`가 분당 한도(429)·동시요청(409) 제한을 우회한다. 글자수 상한은 별도 토글 없이 기존 `AI_CHAT_MAX_MESSAGE_LENGTH`로 조절(중복 제거), 빈 입력 검증(400)은 항상 유지. `AiChatRateLimiterTest` +1건, 전체 백엔드 79건 통과. (커밋 `d64369f`)
8. ✅ **로그/진단 정리 (완료, 2026-06-21)** — `SimpleLoggerAdvisor`는 기본 미등록·logger `OFF`로 두고, 명시적으로 활성화해도 질문·답변 원문 대신 메시지 수·결과 수·Tool 호출 여부·token 사용량만 기록하도록 변경. `HouseTools` DEBUG 로그도 조회 조건·결과 원문을 제거하고 집계 건수만 기록한다. 개발·운영 모두 원문 로그 금지, 운영 장애 시 승인된 제한 시간 동안 안전한 메타데이터만 활성화하는 [로그 및 개인정보 보호 정책](../reports/2026-06-21-ai-logging-privacy-policy.md)을 확정했다. `AiConfigTest` 3건과 Tool 로그 비노출 테스트 1건을 추가했으며 전체 백엔드 테스트 72건이 통과했다.
