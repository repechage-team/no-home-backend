## 3. tool calling 혼합형 재설계 (BE #29 / FE #13)
- **무엇**: 분리형(`/chat`+`/agent`, 모드 토글) → **단일 `POST /api/ai/assistant`**. LLM이 tool calling으로 스스로 분기 — 질문이면 데이터 tool→텍스트 답변, 페이지 조작이면 액션 tool→명령. "아무 tool도 안 부르고 일반 답변"이 1급 선택지라 모호·불만 발화가 억지 action으로 새지 않는다.
- **핵심 메커니즘(Phase 0 실측)**: 액션 tool `@Tool(returnDirect=true)`가 반환한 `AgentCommand`가 `.call().content()`에 **JSON으로** 실리고, `finishReason=="returnDirect"`로 결정적 식별 → 컨트롤러가 역직렬화해 `type=command`, 아니면 `type=answer`.
- **백엔드**: `AiAssistantController`(`/assistant`) · `PageActionTools`(6 액션 tool: applyFiltersAndSearch/setFilters/paginate/selectItem/mapFocus/reset, 본문에서 `AgentCommandGuards.validate` 적용) · 공통 유틸 `AiProviderErrors`(timeout/auth 분류)·`AiRequests`(memberId/conversationId) · `HouseTools` 전월세(dealMode) 확장(보증금/월세 통계). 레거시 `AiChatController`/`AiAgentController` 제거.
- **프론트**: `parseAssistantResponse`(answer/command 분기) · `ChatWidget`(단일 호출, 모드 토글 제거) · `filterSchema` 전월세 키 추가(capability drift 해소) + **폼↔filterSchema 동기화 테스트**(drift 재발 차단) · App.vue `handleAgentCommand` 재사용. 검색 폼 기본 거래월을 최신월(직전월, `currentDealMonth`)로 세팅해 거래월 미지정 검색 0건 방지.
- **검증**: backend 144 / frontend 44 그린 + 브라우저 E2E(질문·검색·전세·월세·페이지·불만·비서울·멀티턴). 단기기억은 같은 빈으로 자동 승계.

## 5. 환경 메모
- 디렉터리 표준 `no-home-backend`/`no-home-frontend`/`no-home-artifact`. full-stack: `cd no-home-artifact && MYSQL_PORT=3307 docker compose up --build -d`.
- 호스트 MySQL 3306 점유 → docker MySQL은 **3307** remap(컨테이너 내부 3306, 백엔드는 `mysql:3306`).
- **전월세/라이브 검색용 DB 스키마**: 전월세 컬럼·`public_data_import_batches` 테이블이 추가됨. 기존 docker volume이 구 스키마면 `docker compose down -v` 후 재기동 필요(운영/팀 환경도 마이그레이션 필요).
- AI는 `/api/ai/assistant` 단일 엔드포인트. `SSAFY_GMS_API_KEY` 미설정 시 ChatClient 빈 없음 → 503 graceful(앱은 정상 기동).
- 작업 규칙은 [2026-06-21 가이드 §5](2026-06-21-handoff.md) 동일: 비밀값 출력·`.env` 커밋·`git add -A` 금지(선별 staging), 편집 전 feature 브랜치, commit까지만 자동(push/PR/merge는 확인).
