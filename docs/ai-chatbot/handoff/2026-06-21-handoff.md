# AI 챗봇 최종 인계 / 재개 가이드

다른 환경(PC)에서 작업을 이어받는 사람을 위한 인계 문서. 이 PC 전용 정보는 담지 않으며, 어느 환경에서도 동일하게 따라갈 수 있다.

## 1. 기준선 (master에 머지 완료)
- **backend**: Spring Boot 3.5.9 + Spring AI 1.1.2. `POST /api/ai/chat`(로그인 전용, `/api/ai/**` 인터셉터). `HouseTools`가 기존 `HouseService.searchHouseDeals`를 재사용해 서울 실거래가 조회 — `dealYmd`(YYYYMM·2006~현재·미래불가) 검증, 자동 임포트 실패 시 친화 폴백, 연월 미지정 시 DB우선+되묻기. 사용자별 사용량 리미터, 오류 매핑(400/401/409/429/503/504), 원문 비기록 로깅.
- **frontend**: `ChatWidget.vue`(플로팅 위젯, 접근성 label/aria-live, Unicode code-point 길이) + 순수 로직 `src/chat/chatClient.js`(node:test).
- **데이터**: `data.sql`에 `SET NAMES utf8mb4;` — 한글 시드 더블 인코딩 해결.
- 관련 PR(머지됨): backend feat(ai)/chore+fix+docs, frontend feat(chat).

## 2. 새 환경 셋업
1. clone(`master`): `no-home-backend`, `no-home-frontend`, `no-home-artifact`(통합 실행/문서).
2. `.env`: 각 레포 `.env.example` → `.env` 복사 후 **본인 키 입력**(`SSAFY_GMS_API_KEY`, `PUBLIC_DATA_SERVICE_KEY`, `KAKAO_MAP_API_KEY`/`VITE_KAKAO_MAP_API_KEY`). 비밀값은 새 PC에서 직접 입력(커밋·공유 금지).
3. frontend `npm install`; backend `docker compose up -d mysql`(한글 시드 정상화를 위해 최초 1회 `down -v && up` 권장).
4. 그린 확인: backend `./mvnw test`, frontend `npm test` + `npm run build`.

> git으로 전달되지 않는 것(정상): 로컬 `.env`(실키), `node_modules`/`target`(재생성). 비밀값은 git으로 옮기지 않는다.

## 5. 작업 규칙
- 비밀값 출력·`.env` 커밋·`git add -A` 금지(그룹별 선별 staging).
- `no-home-artifact`의 기존 문서는 이전 오너의 온보딩 자료(미합의 규약) — 새 팀 합의 전 구조 변경/이관하지 않는다.
- 작업 단위는 작은 PR로 분리, 보안 P0(이력·키)는 push/배포 전 결론을 낸다.
