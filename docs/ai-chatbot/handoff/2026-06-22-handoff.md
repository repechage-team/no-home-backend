## 2. 에이전트 모드 MVP (오늘 머지: BE #16 / FE #5)
- **무엇**: 질문 모드(`/api/ai/chat`)는 유지, **실행 모드** 추가. 사용자는 대화만 하고 백엔드가 자연어→구조화 명령(`AgentCommand`)을 반환, 프론트가 검색 페이지를 조작(필터 채움→검색→사후 요약).
- **백엔드**: `POST /api/ai/agent`(`com.ssafy.home.ai.agent`/`controller`). `.entity(AgentCommand)` temp 0, capability 주입 프롬프트, `AgentCommandGuards`(action·지역·연월 가드→clarify), `@Nullable ChatClient` 503·리미터·auth/timeout 재사용, 파싱/가드 실패는 200 clarify.
- **프론트**: `houseSearchParams.js`의 `filterSchema`/`capabilities`/`applyAgentFilters`(★ capability 단일 출처, 미인식 키 무시·보고), `chat/agentClient.js`, `ChatWidget.vue`(질문/실행 토글+emit), `App.vue`(`handleAgentCommand`).
- **범위**: MVP=액션 `search·setFilters·reset·clarify` + 필터 `지역·날짜·아파트명`. 상세·API 계약·검증은 [구현 리포트](../reports/2026-06-22-agent-mode-mvp-implementation.md).
- **검증**: 백엔드 105 / 프론트 28 그린 · API E2E 9/9 · 실제 브라우저 스크린샷(모드 토글·검색 자동실행·clarify).

## 4. 환경 메모 (당일 검증에서 확인)
- 디렉터리는 **팀 표준 `no-home-backend`/`no-home-frontend`/`no-home-artifact`**. full-stack은 `cd no-home-artifact && MYSQL_PORT=3307 docker compose up --build -d`.
- 호스트 MySQL이 3306 점유 → docker MySQL은 **3307**로 remap(컨테이너 내부는 3306, 백엔드는 docker 네트워크 `mysql:3306`).
- 공공데이터 키 미적재 시 검색이 0건/실패로 보일 수 있음(에이전트·챗 동작과 무관, 데이터 계층 이슈). Kakao 지도 키 없으면 지도 SDK 로드 실패(페이지 구조는 유지).
- 작업 규칙은 [2026-06-21 가이드 §5](2026-06-21-handoff.md) 동일: 비밀값 출력·`.env` 커밋·`git add -A` 금지(선별 staging), 작은 PR 분리.
