# 트러블슈팅: AI 응답/프롬프트 — "환각"처럼 보인 인코딩 문제

## 증상
`/api/ai/chat`가 "동작구 2024년 5월" 질의에 대해 **숫자(건수·평균/최저/최고가)는 정확**한데
**대표 거래의 아파트명은 매 호출 다르게 엉뚱**하게 나옴.
- 1차: "래미안신반포", "신대방삼거리" …
- temperature 0 적용 후에도: "래미안신반포리오센트", "신반포센트럴아이파크", 심지어 '신반포동'(동작구에 없음)

## 진단 과정
1. **로그에 툴 호출 확인**: `Executing tool call: searchSeoulAptDeals` / `Successful execution` → 툴은 정상 호출됨.
2. **프롬프트/온도 보강 시도**(아래) → 그래도 이름 환각 지속. "단순 LLM 환각" 가설 약화.
3. **툴 결과를 서버 로그로 직접 출력**(`HouseTools`에 `log.debug`) →
   ```
   대표 거래:
   - ì•„í¬ë¡œë¦¬ë²„í•˜ìž„ (í‘ì„ë™), 20억 5,000만원 ...   ← DB에서 읽은 이름이 이미 깨짐
   ```
   정적 한글(총 거래 건수/평균 거래가 등)은 멀쩡, **DB에서 읽은 값만 mojibake**.
4. 결론: **모델이 깨진 입력을 받아 그럴듯한 한글 이름으로 "복원"** 한 것. 숫자는 ASCII라 손상 없이 통과.
   → 근본 원인은 LLM이 아니라 데이터 인코딩. [ISSUE: seed-data 더블 인코딩](../../_shared/issues/2026-06-21-seed-data-double-encoding.md)으로 분리.

## 적용한 보강 (유지)
LLM 답변 충실도를 위한 합리적 설정이라 인코딩 수정과 별개로 유지:
- `AiChatController` 시스템 프롬프트에 고유명사 보존 규칙 추가: *"아파트명·동명 등 고유명사는 도구 결과의 글자를 절대 바꾸지 말고 그대로 복사하라."*
- 사실형 Q&A이므로 `temperature 0` 설정(`ChatOptions.builder().temperature(0.0)`).
- 추측 금지/서울만 지원/없으면 없다고 답하기 규칙.

> 단, 보강만으로는 해결 안 됨이 확인됨 — **입력 데이터가 깨지면 프롬프트로 못 막는다.** 인코딩 이슈 수정이 선행되어야 이름이 정확해짐.

## 교훈
- LLM "환각"처럼 보여도 **모델에 들어가는 입력(툴 결과)부터 검증**하되, 운영 로그에는 원문을 남기지 않는다. 테스트 fixture나 접근이 통제된 로컬 디버거로 확인한다.
- 숫자는 맞고 고유명사만 틀리면 인코딩 손상을 의심하라.
- `SimpleLoggerAdvisor`는 기본 미등록·logger `OFF`이며, 명시적으로 활성화해도 원문 대신 메시지 수와 token 사용량만 기록한다. 세부 기준은 [로그 및 개인정보 보호 정책](../reports/2026-06-21-ai-logging-privacy-policy.md)을 따른다.
