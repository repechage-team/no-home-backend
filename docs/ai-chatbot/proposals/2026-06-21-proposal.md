# AI 챗봇 도입 제안 & 백엔드 버전업 협의 요청

대상: no-home 백엔드 팀  ·  상태: **✅ 확정 (팀 합의 완료)**

> **확정 결과**
> - 버전업 진행 확정 (옵션 A: 기존 백엔드 통합 + Boot 3.5.x)
> - 챗봇 용도: A(부동산 질의) → C(대화기억) 단계적 확장
> - 챗봇 인증: **로그인 사용자만** (`/api/ai/**` 인터셉터 보호)
> - 비밀키: 기존 키를 그대로 `.env`로 이전(코드 하드코딩 금지)

## 1. 한 줄 요약
no-home에 "부동산 질의 AI 챗봇"을 추가하려 합니다. 이를 위해 **백엔드 Spring Boot를
3.3.5 → 3.5.x로 올려야** 하며(Spring AI 1.1.x 요구사항), 이는 공유 빌드 변경이라 팀 합의를 요청드립니다.

## 2. 무엇을 만드나
- 사용자가 자연어로 아파트 실거래가를 질문하면(예: "강남구 2024년 5월 평균 거래가?"),
  AI가 `@Tool`을 통해 기존 `house_deals` DB를 조회해 답변.
- 사내 학습 프로젝트 `spring-ai-project_lab`의 Tool 호출 에이전트 구조를 이식(스택 동일: Java17/MyBatis/OpenAI-GMS).
- 프론트는 Vue에 플로팅 챗 위젯 추가. 신규 패키지 `com.ssafy.home.ai`로 격리.

## 3. 왜 버전업이 필요한가
- 현재 백엔드: Spring Boot **3.3.5**.
- Spring AI 1.1.x는 Spring Boot **3.4+** 필요. → 통합하려면 버전업 불가피.

## 4. 버전업 영향도 점검 결과 (코드 실측, 리스크 낮음)
| 구분 | 결과 |
|---|---|
| 컨트롤러/서비스 테스트 | 순수 Mockito + `standaloneSetup` → `@MockBean` 미사용, 영향 없음 |
| 웹/HTTP API | `WebMvcConfigurer`, `RestClient`, `jakarta.servlet` 전부 3.5 호환 |
| 인증(JWT) | JDK crypto 기반, Boot 독립. Security 오토컨피그 미사용 |
| 설정 키 | datasource/hikari/mybatis/encoding 등 3.5에서 유효 |
| Java 버전 | **17 유지** (상향 불필요) |
| **필수 변경** | `mybatis-spring-boot-starter` 3.0.4 → **3.0.5** + parent 버전 |
| 검증 카나리아 | `MemberMapperTest`(@MybatisTest+H2) 통과로 호환 확인 |

→ 실제 변경은 `pom.xml` 2줄 수준. `spring-ai-project_lab`이 Boot 3.5.9+mybatis 3.0.5로 이미 검증.

## 5. 결정 결과
- [x] (A) 기존 백엔드에 통합 + Boot 3.5.x 버전업 진행  ← **확정**
- [ ] (B) 또는 AI를 별도 마이크로서비스로 분리 — 미채택
- [x] 버전업: 별도 브랜치에서 전체 회귀 테스트 후 머지 (Boot 3.5.9 / Spring AI 1.1.2 / mybatis 3.0.5)
- [x] 챗봇 인증 정책: **로그인 전용**
- [x] 챗봇 용도: A(부동산 질의) → C(대화기억) 단계적 확장

## 6. 진행 절차(합의 후)
1) 별도 브랜치에서 parent 3.5.x + mybatis 3.0.5 적용 → `./mvnw clean test` 전체 통과 확인
2) `com.ssafy.home.ai` 패키지로 Spring AI/챗 엔드포인트 추가
3) 프론트 챗 위젯 연동
